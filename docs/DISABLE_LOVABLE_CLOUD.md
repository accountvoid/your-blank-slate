# تعطيل Lovable Cloud للمشاريع المستقبلية

> ⚠️ **هام:** لا يمكن إزالة Lovable Cloud من هذا المشروع بعد تفعيله. حتى الرجوع لإصدار سابق لن يزيله. الخطوات أدناه تعطّله فقط **للمشاريع الجديدة** التي ستنشئها لاحقاً.

---

## 1. تعطيل Cloud للمشاريع المستقبلية

من دون كسر المشروع الحالي:

1. افتح الشريط الجانبي في Lovable.
2. اذهب إلى **Connectors** (في الجذر، وليس داخل إعدادات المشروع).
3. اختر **Lovable Cloud**.
4. اضغط **Disable Cloud**.

النتيجة:
- ✅ المشروع الحالي يستمر بالعمل بشكل طبيعي (لا يتأثر).
- ✅ أي مشروع جديد تنشئه لاحقاً سيبدأ بدون Cloud.
- ✅ ستحتاج لربط Supabase يدوياً في المشاريع الجديدة.

---

## 2. ربط مشروع جديد بـ Supabase الخاص بك

بعد تعطيل Cloud وإنشاء مشروع جديد:

1. افتح المشروع الجديد في Lovable.
2. اضغط زر **Supabase** في أعلى يمين المحرر.
3. اختر **Connect Supabase**.
4. سجّل الدخول لحسابك واختر المشروع الذي تريد ربطه.
5. Lovable ستضبط `VITE_SUPABASE_URL` و `VITE_SUPABASE_PUBLISHABLE_KEY` تلقائياً.

---

## 3. ماذا يحدث عند عمل Remix لهذا المشروع؟

⚠️ **تحذير:** الـ Remix **يرث ربط Lovable Cloud** تلقائياً — لا يوجد طريقة برمجية لإجباره على استخدام Supabase خارجي.

### الحل اليدوي بعد الـ Remix:

بما أن Cloud لا يمكن إزالته من نسخة الـ Remix أيضاً، لديك خياران:

**الخيار أ — إبقاء Cloud (الأسهل):**
- استخدم Cloud الجديد الذي تم إنشاؤه للنسخة (سيكون مشروع Supabase منفصل تديره Lovable).

**الخيار ب — استخدام Supabase الخاص بك (يتطلب عمل يدوي):**
1. لا تستطيع تعديل `.env` أو `src/integrations/supabase/client.ts` مباشرة (سيتم استبدالها).
2. بدلاً من ذلك: أنشئ مشروعاً فارغاً جديداً في workspace الآخر **بعد** تعطيل Cloud (الخطوة 1)، ثم انسخ الكود يدوياً من:
   - `src/` (كل المكونات والصفحات والـ hooks)
   - `supabase/migrations/` (شغّلها في مشروع Supabase الخاص بك)
   - `supabase/functions/` (انسخ الـ Edge Functions)
   - `public/`، `package.json`، `tailwind.config.ts`، `vite.config.ts`
3. اربط Supabase الخاص بك من زر **Connect Supabase** في المحرر.

---

## 4. ملاحظات مهمة

- **مفاتيح Cloud الحالية (`VITE_SUPABASE_*`) لا تلمسها.** إفراغها أو استبدالها يدوياً يكسر التطبيق عند النشر.
- **الأسرار** (مثل `NOWPAYMENTS_API_KEY`) مخزّنة في Cloud وليست في `.env`. عند الانتقال إلى Supabase الخاص بك، ستحتاج لإعادة إضافتها في لوحة Supabase → Edge Functions → Secrets.
- **الهجرات (Migrations)** الموجودة في `supabase/migrations/` قابلة للتشغيل على أي مشروع Supabase — هذه هي أفضل نقطة انطلاق للنقل.
- **الـ RLS Policies و Functions و Triggers** كلها موجودة في ملفات الهجرات، ستنتقل معك.

---

## 5. ملخص القرار

| الحالة | ما يحدث |
|--------|---------|
| هذا المشروع الحالي | يبقى على Lovable Cloud (لا يمكن إزالته) |
| مشاريع جديدة بعد Disable Cloud | تبدأ فارغة، تربطها بـ Supabase يدوياً |
| Remix لهذا المشروع | يرث Cloud — لا يمكن تغيير ذلك برمجياً |
| نقل حقيقي لـ Supabase خاص | يتطلب إنشاء مشروع فارغ + نسخ الكود يدوياً |
